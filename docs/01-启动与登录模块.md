# 01 - 启动与登录模块 📖

> 📌 本文档详解程序从启动到登录成功的完整流程，包含每一行关键代码的含义。
> 适合完全没有项目经验的开发者阅读。

---

## 一、这个模块是干什么的？

💡 **用大白话说**：就像你打开手机上的 APP，先看到登录界面，输入账号密码，点登录，进去主界面——这个模块负责的就是这个过程。

**具体职责**：
1. 启动程序 → 初始化网络和 FFmpeg
2. 弹出登录窗口 → 输入账号密码
3. 连接服务器 → 发送登录/注册请求
4. 登录成功 → 进入视频浏览主界面
5. 登出 → 回到登录窗口（可以换号登录）

## 二、程序启动流程（从 main.cpp 开始）

### 🎬 就像开公司：先租办公室，再招人，最后开门营业

```
程序启动
  │
  ├── 1. 创建 QApplication（租办公室 —— 建立程序运行环境）
  ├── 2. 注册 QImage 类型（办工牌 —— 让信号槽能传递图片）
  ├── 3. 初始化 FFmpeg 网络（装网线 —— 让 FFmpeg 能播网络视频）
  ├── 4. 创建 NetworkManager（招网络经理 —— 负责所有网络通信）
  │
  └── 5. 进入主循环（开门营业）
        │
        ├── 弹出登录窗口
        │     ├── 登录成功 → 进入主界面
        │     └── 关闭窗口 → 程序退出
        │
        └── 主界面运行中
              ├── 正常关闭 → 程序退出
              └── 点"登出" → 回到登录窗口
```

### 逐行解析 main.cpp

```cpp
#include "logindialog.h"      // 【引入】登录对话框的头文件
#include "videomainwindow.h"  // 【引入】主窗口的头文件
#include "networkmanager.h"   // 【引入】网络管理器的头文件

#include <QApplication>       // 【引入】Qt 应用程序类，每个 Qt 程序必须有
#include <QImage>             // 【引入】图片类，用于跨线程传递视频帧

extern "C" {                  // 【说明】FFmpeg 是 C 语言写的，用 C++ 编译器编译时
#include "libavformat/avformat.h"  // 需要用 extern "C" 告诉编译器按 C 规则链接
}

#undef main                   // 【说明】SDL2 会把 main 宏替换成 SDL_main，
                             // 用 #undef main 取消这个替换，让 Qt 的 main 生效

int main(int argc, char *argv[])
{
    QApplication a(argc, argv);          // 【步骤1】创建 Qt 应用程序对象
                                         // argc/argv 是命令行参数
                                         // QApplication 管理整个 GUI 的生命周期

    qRegisterMetaType<QImage>("QImage"); // 【步骤2】注册 QImage 元类型
                                         // 原因：Qt 的信号槽跨线程传递自定义类型时，
                                         // 需要提前注册，否则运行时报错
                                         // 视频播放器的视频帧就是通过信号槽传递 QImage

    avformat_network_init();             // 【步骤3】初始化 FFmpeg 网络模块
                                         // 作用：让 FFmpeg 能读取网络上的视频流
                                         // 比如打开 http://xxx/video.m3u8 这样的地址
                                         // 如果不调用这个，avformat_open_input 打不开网络 URL

    NetworkManager networkManager;       // 【步骤4】创建网络管理器
                                         // 这个对象负责：
                                         // - 连接/断开服务器
                                         // - 发送所有业务消息（登录、注册、上传等）
                                         // - 接收并解析服务器响应
                                         // 整个程序只有一个实例，多个窗口共享

    while (true) {                       // 【步骤5】主循环 —— 支持"登出后重新登录"
                                         // 正常退出程序 → break 跳出循环
                                         // 用户点"登出" → 回到循环开头，重新弹出登录窗口

        LoginDialog loginDialog(nullptr, &networkManager);
                                         // 【步骤5a】创建登录对话框
                                         // 参数1: parent = nullptr，没有父窗口
                                         // 参数2: &networkManager，传入网络管理器的指针
                                         //        登录窗口需要用它来发送登录请求

        if (loginDialog.exec() != QDialog::Accepted)
            break;                       // 【步骤5b】exec() 是模态显示 —— 阻塞等待用户操作
                                         // 返回值：QDialog::Accepted = 登录成功（点登录后验证通过）
                                         //         QDialog::Rejected = 关闭窗口（点取消或X）
                                         // 如果不是 Accepted，break 跳出循环，程序退出

        VideoMainWindow w(&networkManager);
                                         // 【步骤5c】创建主窗口
                                         // 传入同一个网络管理器，主窗口用它来请求视频列表等

        bool loggedOut = false;          // 【步骤5d】标记用户是否主动登出
                                         // false = 正常关闭程序
                                         // true = 用户点了"登出"

        QObject::connect(&w, &VideoMainWindow::SIG_logout, [&]() {
            networkManager.setToken(""); // 清空 token（相当于"注销登录态"）
            loggedOut = true;            // 标记为"登出"
            w.close();                   // 关闭主窗口，a.exec() 返回
        });                              // 【说明】lambda 捕获了 loggedOut 的引用

        w.show();                        // 【步骤5e】显示主窗口（非模态，不会阻塞）

        a.exec();                        // 【步骤5f】进入 Qt 事件循环
                                         // 程序在这里"停住"，等待用户操作
                                         // 用户关闭窗口或点登出 → exec() 返回

        if (!loggedOut)
            break;                       // 【步骤5g】如果不是登出，说明是正常关闭
                                         // 直接 break，程序退出
                                         // 如果是登出，不 break，回到 while 开头
                                         // 重新创建登录窗口
    }

    return 0;                            // 程序正常退出
}
```

## 三、登录对话框（LoginDialog）

### 🎬 就像银行的柜台窗口：你递交材料，工作人员帮你办业务

**LoginDialog 的职责**：
- 显示登录界面（用户名 + 密码 + 登录/注册/取消按钮）
- 校验用户输入
- 触发网络连接和登录请求
- 根据登录结果显示提示或跳转

### 3.1 构造函数

```cpp
LoginDialog::LoginDialog(QWidget *parent, NetworkManager* networkManager)
    : QDialog(parent)                    // 【初始化】调用父类 QDialog 的构造函数
                                         // parent = 父窗口，nullptr 表示独立窗口
    , ui(new Ui::LoginDialog)            // 【初始化】创建 UI 对象
                                         // Qt 的 .ui 文件会被编译成 Ui::LoginDialog 类
                                         // 里面定义了界面上的所有控件（按钮、输入框等）
    , m_networkManager(networkManager)    // 【初始化】保存网络管理器指针
                                         // 后续发送登录请求都要用这个指针
    , m_isConnecting(false)              // 【初始化】标记是否正在连接服务器
                                         // 防止用户疯狂点登录按钮导致重复连接
{
    ui->setupUi(this);                   // 【步骤1】初始化 UI 控件
                                         // 把 .ui 文件中设计的界面应用到当前窗口
                                         // 之后就可以通过 ui->le_username 等访问控件

    this->setWindowTitle("短视频播放平台 - 登录"); // 【步骤2】设置窗口标题
    this->setFixedSize(400, 300);        // 【步骤3】设置窗口固定大小 400×300 像素
                                         // 固定大小 = 用户不能拖拽改变窗口尺寸

    ui->le_password->setEchoMode(QLineEdit::Password);
                                         // 【步骤4】密码框设置为密码模式
                                         // 输入的内容显示为 ●●●●，防止旁人看到密码

    initNetwork();                       // 【步骤5】初始化网络信号槽连接
}
```

### 3.2 initNetwork() — 连接信号和槽

💡 **信号槽就像微信**：
- 信号（Signal）= 发微信消息："我登录成功了！"
- 槽（Slot）= 收到消息后要做的事："跳转到主界面"

```cpp
void LoginDialog::initNetwork()
{
    if (!m_networkManager) {             // 【检查】如果网络管理器是空指针
        QMessageBox::critical(this, "错误", "网络管理器未初始化！");
        return;                          // 弹出错误提示，直接返回
    }

    // 【步骤1】先断开旧连接
    // 为什么？因为 LoginDialog 可能在 while 循环中被多次创建
    // 如果不断开旧连接，每次创建都会 connect 一次，导致信号触发多次
    disconnect(m_networkManager, &NetworkManager::SIG_connected,
               this, &LoginDialog::slot_connected);
    disconnect(m_networkManager, &NetworkManager::SIG_loginResult,
               this, &LoginDialog::slot_loginResult);
    disconnect(m_networkManager, &NetworkManager::SIG_error,
               this, &LoginDialog::slot_error);

    // 【步骤2】重新建立连接
    connect(m_networkManager, &NetworkManager::SIG_connected,
            this, &LoginDialog::slot_connected);
    // 【含义】当网络管理器发出"连接成功"信号时，调用 slot_connected 函数
    // 场景：用户点登录 → 连接服务器 → 服务器响应 → 触发这个信号

    connect(m_networkManager, &NetworkManager::SIG_loginResult,
            this, &LoginDialog::slot_loginResult);
    // 【含义】当收到"登录结果"信号时，调用 slot_loginResult 函数
    // 场景：服务器返回"登录成功"或"密码错误"时触发

    connect(m_networkManager, &NetworkManager::SIG_error,
            this, &LoginDialog::slot_error);
    // 【含义】当网络出错时，调用 slot_error 函数
    // 场景：连接超时、服务器断开等
}
```

### 3.3 validateInput() — 校验用户输入

```cpp
bool LoginDialog::validateInput()
{
    QString username = ui->le_username->text().trimmed();
    // 【说明】从用户名输入框获取文本
    // .trimmed() 去掉首尾空格（防止用户不小心输入空格）

    QString password = ui->le_password->text();
    // 【说明】从密码输入框获取文本
    // 注意：密码不去 trimmed()，因为密码中间可能真有空格

    if (username.isEmpty()) {            // 用户名为空
        QMessageBox::warning(this, "提示", "请输入用户名");
        ui->le_username->setFocus();     // 光标回到用户名输入框，方便用户直接输入
        return false;                    // 校验失败
    }

    if (password.isEmpty()) {            // 密码为空
        QMessageBox::warning(this, "提示", "请输入密码");
        ui->le_password->setFocus();
        return false;
    }

    return true;                         // 校验通过
}
```

### 3.4 on_pb_login_clicked() — 点击登录按钮

💡 **这个函数名有讲究**：`on_pb_login_clicked` 是 Qt 的**自动连接**命名规则。
- `on_` + 控件名(`pb_login`) + `_clicked` = 按钮点击时自动调用
- `pb_login` 是 .ui 文件中登录按钮的 objectName

```cpp
void LoginDialog::on_pb_login_clicked()
{
    if (!validateInput()) {              // 【步骤1】先校验输入
        return;                          // 输入不合法，直接返回，不往下执行
    }

    if (m_isConnecting) {                // 【步骤2】防重复点击
        return;                          // 正在连接中，忽略重复点击
    }

    // 【步骤3】如果已经连接到服务器，直接发登录请求
    if (m_networkManager->isSocketConnected()) {
        QString username = ui->le_username->text().trimmed();
        QString password = ui->le_password->text();
        m_networkManager->sendLogin(username, password);
        // 直接发送，不需要再连接
        return;
    }

    // 【步骤4】还没连接服务器，先连接
    m_isConnecting = true;               // 标记为"正在连接"
    setUiEnabled(false);                 // 禁用界面控件（防止用户乱点）
    m_networkManager->connectToServer(); // 发起 TCP 连接
    // 连接成功后，slot_connected() 会被自动调用
}
```

### 3.5 登录成功/失败的完整数据流

```
用户输入账号密码，点登录
    │
    ▼
on_pb_login_clicked()
    │  validateInput() 校验通过
    │
    ├─── 已经连接? ──→ 直接调用 sendLogin()
    │
    └─── 没连接? ──→ connectToServer() → TCP连接中...
                         │
                         ▼ (TCP连接成功)
                    slot_connected()
                         │
                         ▼
                    sendLogin(username, password)
                         │
                         ▼ (密码经过 authPasswordHash 加密)
                    sendMessage() → 构造二进制帧 → 发送到服务器
                         │
                         ▼ (服务器处理登录)
                    服务器返回响应
                         │
                         ▼
                    onReadyRead() → parseMessage() → dispatchResponse()
                         │
                         ▼ (action == "login")
                    emit SIG_loginResult(success, message, token)
                         │
                         ▼
                    slot_loginResult(success, message, token)
                         │
                    ┌────┴────┐
                    │         │
               成功=true   失败=false
                    │         │
              accept()    QMessageBox 弹出错误提示
              (关闭对话框,
               回到 main.cpp
               进入主窗口)
```

### 3.6 slot_loginResult() — 处理登录结果

```cpp
void LoginDialog::slot_loginResult(bool success, const QString& message, const QString& token)
{
    setUiEnabled(true);                  // 恢复界面控件可用

    if (success) {
        // 登录成功
        emit SIG_loginSuccess(ui->le_username->text().trimmed(), token);
        // 发出"登录成功"信号，携带用户名和 token
        // token 是服务器返回的会话凭证，后续请求可能要用

        this->accept();                  // 关闭对话框，返回值 = QDialog::Accepted
                                         // 这会让 main.cpp 中的 loginDialog.exec() 返回 Accepted
                                         // 然后创建 VideoMainWindow 主窗口
    } else {
        // 登录失败
        QString errMsg = message.isEmpty() ? "登录失败，请重试" : message;
        // 防止 message 为空导致弹框没有文字

        QMessageBox::warning(this, "登录失败", errMsg);
        // 弹出警告框，告诉用户为什么失败
        // 可能的原因："用户不存在" / "密码错误"
    }
}
```

## 四、注册对话框（RegisterDialog）

### 🎬 和登录对话框结构几乎一样，只是多了"确认密码"和更严格的校验

### 4.1 注册流程

### 4.2 validateInput() — 注册的输入校验更严格

```cpp
bool RegisterDialog::validateInput()
{
    QString username = ui->le_username->text().trimmed();
    QString password = ui->le_password->text();
    QString confirmPassword = ui->le_confirmPassword->text();
    // 【说明】比登录多了一个"确认密码"字段

    if (username.isEmpty()) {            // 用户名不能为空
        QMessageBox::warning(this, "提示", "请输入用户名");
        ui->le_username->setFocus();
        return false;
    }

    if (username.length() < 3 || username.length() > 20) {
        // 【新增】用户名长度限制 3-20 个字符
        // 太短容易被穷举，太长不好记
        QMessageBox::warning(this, "提示", "用户名长度应在3-20个字符之间");
        ui->le_username->setFocus();
        return false;
    }

    if (password.isEmpty()) {            // 密码不能为空
        QMessageBox::warning(this, "提示", "请输入密码");
        ui->le_password->setFocus();
        return false;
    }

    if (password.length() < 6) {         // 【新增】密码至少 6 位
        QMessageBox::warning(this, "提示", "密码长度不能少于6个字符");
        ui->le_password->setFocus();
        return false;
    }

    if (password != confirmPassword) {   // 【新增】两次密码必须一致
        QMessageBox::warning(this, "提示", "两次输入的密码不一致");
        ui->le_confirmPassword->setFocus();
        return false;
    }

    return true;
}
```

## 五、密码加密（auth_util.h）

### 🎬 就像寄快递：不会把贵重物品裸着寄，要加锁包装

### 5.1 为什么密码要加密？

⚠️ **绝对不能明文传输密码！** 如果网络被监听，明文密码直接泄露。

本项目的做法：**客户端加密 → 服务器只存加密后的哈希值**。

### 5.2 authPasswordHash() — 逐行解析

```cpp
inline QString authPasswordHash(const QString &password)
// 【函数说明】把明文密码加密成 SHA-256 哈希值
// 【参数】password = 用户输入的明文密码，如 "123456"
// 【返回值】64 位十六进制字符串，如 "a3f2b8c1d4e5f6..."
// 【inline】内联函数，编译时直接嵌入调用处，没有函数调用开销

{
    const QString salted = QStringLiteral("ShortVideo_2026_")
        + password + QStringLiteral("_@platform");
    // 【步骤1】加盐（Salt）
    //
    // 加盐 = 在密码前后加一段固定的字符串，增加破解难度
    // 最终拼接结果：  "ShortVideo_2026_123456_@platform"
    //                      ↑ 前盐        ↑密码    ↑ 后盐
    //
    // 💡 为什么加盐？如果只用 SHA-256("123456")，
    // 攻击者用"彩虹表"（预计算的哈希对照表）一秒就能反查出密码。
    // 加了盐之后，彩虹表就失效了，因为"ShortVideo_2026_123456_@platform"
    // 不在任何公开的彩虹表里。
    //
    // ⚠️ 盐值是硬编码的，不是每个用户独立的——这是简化实现。
    // 生产环境应该每个用户一个随机盐，存在数据库里。

    return QString(QCryptographicHash::hash(
        salted.toUtf8(), QCryptographicHash::Sha256).toHex());
    // 【步骤2】计算 SHA-256 哈希
    //
    // salted.toUtf8()          → 把字符串转成 UTF-8 字节序列（SHA-256 处理的是字节）
    // QCryptographicHash::Sha256  → 使用 SHA-256 算法
    // .toHex()                 → 把 32 字节的二进制结果转成 64 字符的十六进制字符串
    // QString(...)             → 包装成 QString 返回
    //
    // 💡 SHA-256 是单向哈希：可以从密码算出哈希，但不能从哈希反推密码。
    // 这就是为什么服务器只存哈希——即使数据库泄露，攻击者也拿不到原始密码。
}
```

### 5.3 密码加密完整流程图

```
用户输入密码: "123456"
       │
       ▼
加盐拼接: "ShortVideo_2026_" + "123456" + "_@platform"
       │
       ▼ 结果: "ShortVideo_2026_123456_@platform"
       │
       ▼
SHA-256 哈希: a3f2b8c1d4e5f6...（64 位十六进制）
       │
       ▼
发送到服务器（JSON 中 "password" 字段的值就是这个哈希）
       │
       ▼
服务器直接存入数据库（不再二次加密）
       │
       ▼
登录时：客户端发来的哈希 == 数据库存的哈希 → 登录成功
        客户端发来的哈希 != 数据库存的哈希 → 密码错误
```

### 5.4 isSha256Hex() — 判断一个字符串是不是合法的 SHA-256 哈希

```cpp
inline bool isSha256Hex(const QString &s)
// 【函数说明】判断字符串是不是 64 位十六进制（SHA-256 的标准格式）
// 【参数】s = 待判断的字符串
// 【返回值】true = 是合法的 SHA-256 哈希格式

{
    if (s.length() != 64)               // SHA-256 输出 32 字节 = 64 个十六进制字符
        return false;                    // 长度不对，肯定不是

    for (const QChar &c : s) {          // 逐个字符检查
        if (c.isDigit())                // 数字 0-9 是合法的
            continue;
        const QChar lc = c.toLower();   // 字母转小写
        if (lc < QLatin1Char('a') || lc > QLatin1Char('f'))
            return false;                // 不是 0-9 也不是 a-f，不合法
    }                                    // SHA-256 的十六进制只包含 0-9 和 a-f

    return true;                         // 全部通过
}
```

### 6.2 connectToServer() — 连接服务器

```cpp
void NetworkManager::connectToServer()
{
    if (m_socket->state() == QAbstractSocket::ConnectedState) {
        emit SIG_connected();            // 已经连接了，直接发"连接成功"信号
        return;                          // 不需要重复连接
    }

    if (m_socket->state() == QAbstractSocket::ConnectingState) {
        qDebug() << "连接正在进行中..."; // 正在连接中，不要重复调用
        return;
    }

    qDebug() << "正在连接服务器：" << m_serverIp << ":" << m_serverPort;
    m_socket->connectToHost(m_serverIp, m_serverPort);
    // 【核心】发起 TCP 连接
    // m_serverIp 和 m_serverPort 从 config.ini 中读取
    // 连接是异步的 —— 不会阻塞等待，连接成功后触发 onConnected()
}
```

### 6.3 loadConfig() — 加载配置

💡 **配置文件查找顺序**：先找当前工作目录，再找程序所在目录。

```cpp
void NetworkManager::loadConfig()
{
    // 【步骤1】优先查找当前工作目录下的 config.ini
    QString configPath = QDir::currentPath() + "/config.ini";
    // QDir::currentPath() = 程序运行时的工作目录
    // 在 Qt Creator 中运行，工作目录通常是项目目录或 build 目录

    if (!QFile::exists(configPath)) {
        // 【步骤2】工作目录没有，找程序所在目录
        configPath = QCoreApplication::applicationDirPath() + "/config.ini";
        // applicationDirPath() = .exe 文件所在的目录
    }

    // 【步骤3】两个地方都没有，创建一个默认配置
    if (!QFile::exists(configPath)) {
        QSettings settings(configPath, QSettings::IniFormat);
        settings.beginGroup("Server");
        settings.setValue("IP", "127.0.0.1");   // 默认连接本机
        settings.setValue("Port", 8888);          // 默认端口 8888
        settings.endGroup();
        settings.sync();                          // 立即写入磁盘
    }

    // 【步骤4】读取配置
    QSettings settings(configPath, QSettings::IniFormat);
    settings.beginGroup("Server");
    m_serverIp = settings.value("IP", "127.0.0.1").toString();
    // 第二个参数是默认值：如果配置文件中没有 IP 字段，就用 127.0.0.1

    m_serverPort = static_cast<quint16>(settings.value("Port", 8888).toUInt());
    // toUInt() 把配置值转成无符号整数
    // quint16 = 无符号 16 位整数，端口号的范围是 0-65535
    settings.endGroup();

    qDebug() << "加载服务器配置（" << configPath << "）：" << m_serverIp << ":" << m_serverPort;
}
```

**config.ini 文件示例**：
```ini
[Server]
IP=10.56.5.9
Port=8888
```

## 七、常见问题

### Q: 为什么要用 `while(true)` 循环包裹登录和主窗口？
> 因为要支持"登出后重新登录"。如果不用循环，登出后程序就退出了。用循环的话，登出 → 关闭主窗口 → 回到登录窗口 → 可以换号登录。

### Q: 为什么 `disconnect` 之后再 `connect`？
> 因为 `LoginDialog` 在 `while` 循环中可能被多次创建和销毁。每次创建都会 `connect`，如果不先 `disconnect`，同一个信号会连接多次，触发时槽函数会被调用多次。

### Q: 密码的盐是硬编码的，安全吗？
> ⚠️ 这是简化实现。生产环境中，盐应该是每个用户独立的随机值，存在数据库里。硬编码盐的风险是：如果盐泄露了，攻击者可以针对这个盐制作彩虹表。但比完全不加盐好得多。

### Q: token 是干什么用的？
> Token 是服务器在登录成功后返回的"会话凭证"。就像酒店房卡——你用身份证（密码）办入住，前台给你一张房卡（token），之后你用房卡而不是身份证来进出。当前项目中 token 的使用还不完善，后续可以用来做请求认证。
