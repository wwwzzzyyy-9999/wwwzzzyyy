## 二、视频卡片（VideoCard）

### 2.1 卡片结构

```
┌────────────────────────┐
│    封面图 (640×360)      │  ← FFmpeg 截取视频第1帧
│                        │
│    ▶ 01:23             │  ← 播放按钮 + 时长
├────────────────────────┤
│ 标题文字                 │  ← video.title
│ 👤 作者 · ▶ 1234次播放  │  ← author + playCount
│ ❤ 56                   │  ← likesCount
└────────────────────────┘
```

### 2.2 关键成员变量

| 变量 | 类型 | 含义 |
|------|------|------|
| `m_videoId` | `int` | 视频 ID（点击时传给播放器） |
| `m_title` | `QString` | 视频标题 |
| `m_coverUrl` | `QString` | 封面图路径 |
| `m_hlsUrl` | `QString` | HLS 播放地址 |
| `m_author` | `QString` | 上传者用户名 |
| `m_playCount` | `int` | 播放量 |
| `m_likesCount` | `int` | 点赞数 |
| `m_duration` | `int` | 时长（秒） |

## 三、VideoMainWindow 核心逻辑

### 3.1 初始化和信号槽连接

```cpp
VideoMainWindow::VideoMainWindow(NetworkManager *networkManager, QWidget *parent)
    : m_networkManager(networkManager)
{
    ui->setupUi(this);

    // 连接网络信号

            this, &VideoMainWindow::slot_videoListResult);

            this, &VideoMainWindow::slot_searchVideosResult);
    // 收到搜索结果 → 刷新卡片网格

    connect(m_networkManager, &NetworkManager::SIG_uploadStartResult,
            this, &VideoMainWindow::slot_uploadStartResult);
    // 上传开始响应 → 记录 uploadId

    connect(m_networkManager, &NetworkManager::SIG_uploadChunkResult,
            this, &VideoMainWindow::slot_uploadChunkResult);

    connect(m_networkManager, &NetworkManager::SIG_uploadEndResult,
            this, &VideoMainWindow::slot_uploadEndResult);
    // 上传完成 → 提示用户

    m_networkManager->sendGetRecommendFeed(0, 20);
```

### 3.2 加载视频列表 — slot_videoListResult()

```cpp
void VideoMainWindow::slot_videoListResult(bool success, const QString &message,
                                            const QJsonArray &videos, quint64 requestId)
{
    if (!success) {
        QMessageBox::warning(this, "提示", message);
        return;
    }

    // 清空旧卡片
    clearVideoCards();

    // 遍历视频数组，创建卡片
    for (const QJsonValue &val : videos) {
        QJsonObject obj = val.toObject();

        VideoCard *card = new VideoCard(this);
        card->setVideoInfo(
            obj["id"].toInt(),           // videoId
            obj["title"].toString(),      // title
            obj["coverUrl"].toString(),   // coverUrl
            obj["hlsUrl"].toString(),     // hlsUrl
            obj["author"].toString(),     // author
            obj["playCount"].toInt(),     // playCount
            obj["likesCount"].toInt(),    // likesCount
            obj["duration"].toInt()       // duration
        );

        // 连接卡片的点击信号
        connect(card, &VideoCard::SIG_playVideo,
                this, &VideoMainWindow::slot_playVideo);

        // 添加到网格布局
        ui->videoGrid->addWidget(card);
    }
}
```

### 3.3 点击视频 → 打开播放器

```cpp
void VideoMainWindow::slot_playVideo(int videoId, const QString &hlsUrl,
                                       const QString &title, const QString &coverUrl)
{
    // 增加播放量
    m_networkManager->sendIncrementPlay(videoId);

    // 打开播放器对话框
    showPlayerDialog(videoId, hlsUrl, title, coverUrl);
}

void VideoMainWindow::showPlayerDialog(int videoId, const QString &hlsUrl,
                                        const QString &title, const QString &coverUrl)
{
    // 使用 QPointer 防止悬空指针
    QPointer<PlayerDialog> dialog = new PlayerDialog(this, m_networkManager);
    dialog->setVideoInfo(videoId, hlsUrl, title, coverUrl);

    // 非模态显示，不阻塞主窗口
    dialog->show();

    // 记录当前播放器，避免重复打开
    m_currentPlayer = dialog;
}
```

## 四、上传流程（客户端侧）

### 4.1 选择文件并开始上传

```cpp
void VideoMainWindow::on_pb_upload_clicked()
{
    QString filePath = QFileDialog::getOpenFileName(this, "选择视频", "", "视频文件 (*.mp4 *.avi *.mkv)");
    if (filePath.isEmpty()) return;

    QFileInfo info(filePath);
    qint64 fileSize = info.size();

    if (fileSize > MAX_FILE_SIZE) {
        QMessageBox::warning(this, "提示", "文件大小超过1GB限制");
        return;
    }

    // 计算分块数量
    int totalChunks = (fileSize + DEFAULT_CHUNK_SIZE - 1) / DEFAULT_CHUNK_SIZE;
    // 向上取整：(99 + 256 - 1) / 256 = 1
    //          (257 + 256 - 1) / 256 = 2

    // 记录文件路径
    m_uploadFilePath = filePath;

    // 发送 UPLOAD_START
    m_networkManager->sendUploadStart(info.fileName(), fileSize, totalChunks);
}
```

### 4.2 收到 UPLOAD_START 响应 → 开始发送数据块

```cpp
void VideoMainWindow::slot_uploadStartResult(bool success, const QString &message,
                                              const QString &uploadId)
{
    if (!success) {
        QMessageBox::warning(this, "上传失败", message);
        return;
    }

    // 记录 uploadId
    m_currentUploadId = uploadId;

    // 开始滑动窗口上传
    QFile file(m_uploadFilePath);
    file.open(QIODevice::ReadOnly);

    // 发送窗口内的前 4 块
    for (int i = 0; i < UPLOAD_WINDOW_SIZE && i < m_currentUploadTotalChunks; i++) {
        QByteArray chunk = file.read(DEFAULT_CHUNK_SIZE);
        m_networkManager->sendUploadChunk(uploadId, i, m_currentUploadTotalChunks, chunk);
        m_uploadWindowBase = i;  // 记录窗口位置
    }
}
```

### 4.3 收到块确认 → 滑动窗口前进

```cpp
void VideoMainWindow::slot_uploadChunkResult(bool success, const QString &message,
                                              int chunkIndex)
{
    if (!success) return;

    // 窗口滑动：发送下一个未发送的块
    int nextChunk = m_uploadNextChunk;  // 下一个要发送的块序号
    if (nextChunk < m_currentUploadTotalChunks) {
        // 读取文件对应位置的数据
        QFile file(m_uploadFilePath);
        file.open(QIODevice::ReadOnly);
        file.seek(nextChunk * DEFAULT_CHUNK_SIZE);  // 定位到对应位置
        QByteArray chunk = file.read(DEFAULT_CHUNK_SIZE);

        m_networkManager->sendUploadChunk(m_currentUploadId, nextChunk,
                                           m_currentUploadTotalChunks, chunk);
        m_uploadNextChunk++;
    }

    // 检查是否全部发送完毕
    if (所有块都已确认) {
        // 计算文件哈希
        QString fileHash = NetworkManager::calculateFileHash(m_uploadFilePath);
        // 发送 UPLOAD_END
        m_networkManager->sendUploadEnd(m_currentUploadId, fileHash);
    }
}
```

## 七、常见问题

### Q: 视频卡片是怎么排版的？
> 用 Qt 的 `QGridLayout` 或 `QScrollArea` + 自定义布局，2 列排列。每个 VideoCard 是一个自定义 QWidget，包含封面图标签、标题标签、信息标签。

### Q: 上传中断了怎么办？
> 客户端可以点击"续传"按钮，调用 `sendUploadResume(uploadId)`，服务端返回已接收的块列表，客户端从断点处继续上传。

### Q: 为什么上传的视频要先转成 HLS？
> 因为播放器用的是 FFmpeg + SDL，对 HLS 流式播放支持最好。直接播 MP4 也可以，但 seek 性能不如 HLS（HLS 每个分片可以独立定位）。
